# 端侧A/B测试轻量化架构方案
## 一、方案定位
本方案定位为**轻量、分层解耦、非侵入、免发版、高性能**的端侧A/B测试架构体系。核心解决端侧A/B测试落地过程中的六大核心痛点：
+ 海量互斥实验导致的性能损耗
+ 多实验冲突与优先级管控难题
+ 用户未登录/登录切换引发的分组漂移
+ 缓存优先策略下的页面UI抖动
+ 配置获取异常时的可靠降级
+ 数据上报与现有埋点体系的融合冲突。

通过标准化分层设计与全链路容错机制，确保架构在满足业务实验灵活迭代需求的同时，极致保障端侧性能、稳定性与用户体验。

## 三、四层核心职责详解
### 1. 配置接入层（Config Layer）
配置接入层是实验配置的入口与处理核心，承担配置精准加载、冲突消解、容错降级三大核心职责，是保障架构轻量化与高可用的关键层。

#### 1.1 两级配置加载机制
摒弃全量拉取的重负载模式，采用**全局公共配置 + 页面级按需配置**的两级拆分加载策略：
- **全局公共基础配置**：应用启动时同步加载，体积极小，仅包含实验元数据（实验ID、路由匹配规则、互斥组标识、优先级配置、全局开关），不携带具体实验参数，核心作用是完成基础元信息校验，不阻塞首屏渲染。
- **页面级业务配置**：采用懒加载策略，仅在页面进入时，根据当前路由按需拉取关联实验的完整配置。切换页面后自动释放对应配置内存，从源头控制端侧内存与网络开销。

#### 1.2 路由自动匹配下发机制
抛弃业务手动订阅的繁琐模式，实现**免订阅、免发版**的路由自动匹配：
- 端侧自动采集当前页面路由（Android/iOS采集页面实例，前端/小程序采集路由Path），无需业务代码额外开发；
- 后台创建实验时，配置路由匹配规则（支持精确匹配、通配符匹配、页面层级匹配），将实验与特定路由绑定；
- 端侧请求配置时，自动携带当前路由，服务端通过规则过滤，仅下发与当前页面匹配的实验配置，避免无关互斥实验冗余下发。

#### 1.3 多实验冲突与优先级管控
针对同一页面/组件多实验共存的场景，通过**分层隔离 + 互斥组管控 + 优先级链**三级机制，确保唯一生效：
- **实验分层隔离**：按业务属性划分为UI展示层、策略逻辑层、算法参数层、营销活动层，不同层级实验可并行生效，同层级实验默认互斥，减少冲突概率。
- **互斥组管控**：为强冲突实验绑定统一互斥组ID，同一互斥组内仅允许一个实验命中，配置层加载时自动过滤组内非生效实验。
- **标准化优先级链**：当同层级/同互斥组存在多实验时，按固定优先级顺序筛选唯一生效实验，优先级从高到低依次为：强制测试白名单 > 全局紧急开关实验 > 自定义业务优先级 > 实验创建时间（最新优先）> 默认对照组。

#### 1.4 异常重试与多级降级
构建完善的容错体系，应对网络抖动、服务不可用等异常场景：
- **轻量化重试策略**：支持1-3次指数退避重试，仅针对网络超时、DNS失败、5xx服务错误等临时异常；4xx永久错误（如参数非法、权限不足）不重试，避免无效请求；
- **多级降级兜底**：当重试全部失败时，按优先级逐级降级：① 本地持久化缓存优先；② 核心页面内置默认基线配置；③ 全部关闭实验，走原生默认版本；
- **异常隔离**：配置模块异常时不抛出错误，不阻塞页面渲染与业务流程，确保核心体验不受影响。

### 2. 流量决策层（Decision Layer）
流量决策层是实验分组分配的核心，负责基于用户标识完成稳定、均匀的分流，并保障分组一致性与渲染非阻塞，是实验体验的核心保障层。

#### 2.1 稳定哈希分流算法
基于唯一标识实现可复现、无抖动的流量分配，核心流程为：
1. **标识选取**：已登录用户优先使用`user_id`，未登录用户使用`device_id`作为分流唯一标识；
2. **哈希计算**：通过确定性哈希算法，对`标识 + 实验ID + 固定盐值`进行运算，取模得到0-99的固定数值，确保同一用户同一实验永远得到同一哈希值；
3. **区间分配**：根据后台配置的实验流量比例（如实验组30%、对照组70%），将哈希值映射至对应区间，完成分组判定，保证流量分布均匀。

#### 2.2 跨登录态分组一致性保障
解决用户未登录/登录切换导致的分组漂移问题，核心机制为：
- **标识绑定继承**：未登录阶段使用`device_id`分流，用户登录成功后，自动将设备维度的实验分组**迁移并绑定至user_id**，后续不再重新哈希计算；
- **本地持久化缓存**：分组结果写入本地持久化缓存，跨会话、跨重启保持一致，确保用户登录前后实验表现与分组完全稳定。

#### 2.3 页面级快照防UI抖动
核心解决缓存优先策略下的页面UI闪烁、跳动问题，核心规则为：
- **页面快照生成**：页面初始化时，基于本地缓存配置生成**当前页面唯一实验快照**，快照包含该页所有实验最终分组结果；
- **生命周期固定**：在页面整个生命周期内（从进入到退出），实验快照不再变更，后台异步更新的新配置仅更新本地缓存，不触发当前页面重渲染；
- **非阻塞渲染**：页面优先使用缓存快照完成渲染，无需等待远程配置更新，彻底避免UI抖动。

#### 2.4 三级缓存与非阻塞渲染
兼顾渲染速度与配置时效性，构建三级缓存体系：
- **一级缓存（内存缓存）**：页面生命周期内，实验变量直接从内存读取，耗时趋近于0，保障快速取值；
- **二级缓存（本地持久化缓存）**：跨会话、跨重启保存，页面初始化优先读取，确保弱网/无网场景下可正常渲染，实现离线可用；
- **三级缓存（远端异步配置）**：后台异步拉取最新配置，仅用于更新本地缓存，不影响当前页面，确保配置最终一致性。

### 3. 变量执行层（Execution Layer）
变量执行层是实验策略的落地载体，承担实验变量解析、业务映射、兜底保障职责，实现对业务层的无侵入接入。

#### 3.1 统一变量获取接口
对外提供简洁、标准化的实验变量获取API，业务层无需感知实验ID、分组、分流逻辑，直接调用接口获取参数即可完成接入，降低开发成本与代码耦合。

#### 3.2 实验策略映射与渲染
根据流量决策层的实验快照结果，将实验分组映射为具体业务内容：
- UI层面：映射文案、样式、布局、组件展示形态；
- 功能层面：映射功能开关、逻辑分支、策略参数；
- 业务层面：映射核心流程中的不同处理路径，满足多版本并行测试需求。

#### 3.3 双层异常兜底与故障隔离
确保实验模块异常不影响主业务流程：
- **第一层兜底**：实验配置异常、分组不存在时，自动返回业务预设默认值，避免空指针、样式缺失等问题；
- **第二层兜底**：实验模块整体异常时，直接走原生默认逻辑，不阻塞页面渲染与功能使用；
- **故障隔离**：实验执行逻辑与主业务解耦，单个实验异常不影响其他实验与核心业务。

### 4. 数据上报层（Report Layer）
数据上报层采用**不独立建设、复用现有埋点**的轻量化设计，避免重复开发与数据冗余，核心实现实验数据与业务数据的统一对齐。

#### 4.1 核心工作流程
1. **实验上下文管理**：维护当前用户全局实验上下文，记录已命中实验、分组及曝光状态，确保同一实验仅记录一次有效曝光；
2. **埋点桥接注入**：埋点系统在初始化或组装事件公共参数时，通过标准接口获取实验上下文，将实验信息（如`ab_experiments: "exp_123:group_a,exp_456:control"`）注入事件公共字段；
3. **业务事件透传**：实验信息随页面浏览、点击、转化等自然业务事件一起上报，无需单独发送实验曝光事件，降低网络开销；
4. **后台数据分析**：服务端日志平台通过实验标识字段解析分组信息，与用户行为、业务指标关联分析，生成实验效果报表。

#### 4.2 全链路复用与保障
- **复用埋点能力**：事件合并、延迟上报、失败重试、本地缓存、加密传输、网络策略等所有可靠性能力，均依赖现有埋点体系，保障数据上报的稳定性与一致性；
- **异常容错**：A/B模块异常时返回空上下文，埋点系统不受影响；埋点系统异常时，A/B模块不感知、不阻塞业务，不影响核心数据上报。

## 五、关键机制总结
### 1. 精准下发机制
**端侧自动路由采集 → 服务端精准过滤 → 仅下发当前页实验**，实现免订阅、免发版的高效配置下发，同时大幅降低端侧性能损耗。

### 2. 冲突解决体系
**实验分层隔离 + 互斥组管控 + 优先级链**，从架构层面彻底解决多实验冲突问题，确保同一组件唯一生效实验。

### 3. 分组稳定保障
**未登录用device_id → 登录后绑定user_id并继承分组**，彻底避免登录态切换导致的分组漂移，保障实验数据与用户体验一致。

### 4. 无抖动渲染方案
**页面快照 + 生命周期固定 + 新配置下页生效**，从根源杜绝缓存优先带来的UI闪烁与跳动，保障用户体验。

### 5. 高可用容错体系
**有限重试 + 三级降级（缓存→内置→默认）**，应对网络、服务等各类异常，确保实验模块永不崩溃、主业务永不阻塞。

### 6. 数据上报融合
**A/B只打标，埋点负责发送**，复用现有埋点体系，无重复建设、无数据冗余，降低维护成本与数据对齐成本。

## 六、方案核心亮点
1. **轻量无侵入**：不依赖重型SDK，模块化集成，不耦合业务代码，接入成本极低；
2. **按需精准加载**：按页面路由精准下发配置，海量互斥实验不影响端侧性能，内存占用可控；
3. **真正免发版**：新增实验仅需在后台配置路由与规则即可上线，无需修改端侧代码、无需打包发版；
4. **分组稳定可追溯**：登录态切换不跳组，可通过user_id/device_id复现用户分组，实验数据可信度高；
5. **零UI抖动**：页面快照机制保障渲染稳定，用户体验无割裂感；
6. **高可用强容错**：完善的重试与降级机制，任何异常不影响主业务流程；
7. **数据链路统一**：复用现有埋点体系，数据上报一致、维护成本低、线上问题易排查。
